## 里程碑 3：以太网调试阻塞（Task #7 部分，硬件搁置）
**时间**：中午
- 按 vendor 参考加 PS GEM0 EMIO + gmii_to_rgmii + RTL8211E PHY
- **核心坑**：vendor PS FCLK_CLK0=200MHz，我们是 50MHz，gmii_to_rgmii 内部 MMCM VCO 算出 250MHz 超出合法范围 → 开 FCLK_CLK2=200MHz 专门喂
- **最终阻塞**：PHY 识别 OK、MDIO 通、autoneg 完成（1000Mbps），但 GEM TX frames 发出去 PC 收不到。试了 RGMII delay 各种 MDIO 页 (0xd08, 0x07)、强制 10/100Mbps、PHY 硬件 reset 拉 H17 高，统统无效
- **结论**：硬件 TX 路径问题（PCB / 磁变 / RJ45），软件层无法修复
- 记忆：`project_luxiaoban_ethernet_blocker.md`

## 里程碑 4：VDMA 帧缓冲 + UART→HDMI 文本控制台（Task #7 完成）
**时间**：下午
- 改用 UART 替代 ETH（POV 最终用 WiFi，当前 PoC 够用）
- 删 VTPG，加 AXI VDMA：PS 写 DDR 0x10000000 帧缓冲，VDMA MM2S 读 DDR 出 AXI Stream
- 新 SmartConnect `axi_smc_hp` 专用 VDMA.M_AXI_MM2S → PS S_AXI_HP0 数据路径
- 全部时钟域统一到 FCLK_CLK1 (75MHz) 避免 VDMA IP 的 `s_axi_lite_aclk ≤ m_axi_mm2s_aclk` 约束违反

- ARM C 写 helloworld.c：5×7 字体渲染 + 滚屏，UART RX 输入 → HDMI 输出
- 记忆：`project_luxiaoban_vdma_framebuffer.md`

## 关键的技术决策 / 踩坑教训

### 时钟
- **不要用 clk_wiz 升 FCLK_CLK0 50M 到 74.25M 给 HDMI**：MMCM 锁不住 → AXI 整个死
- **PS 直出 FCLK_CLK1=75MHz 够跑 720p60**（1% 偏差监视器吃得下）
- **gmii_to_rgmii 需要 200MHz 参考时钟**，不能用 50MHz
- **VDMA IP 约束**：`s_axi_lite_aclk ≤ m_axi_mm2s_aclk`，简单做法全部同一时钟

### 字节序
- AXI Stream TDATA 24-bit RGB: `[7:0]=B, [15:8]=G, [23:16]=R`
- DDR 帧缓冲写入: `fb[offset+0]=B, [+1]=G, [+2]=R`（不是 RGB！）

### xiltimer
- Vitis 2024.2 的 xiltimer.cmake 有 `"Default;"` bug，FSBL build 失败报 undefined reference
- 修复：bsp.yaml 里 `XILTIMER_sleep_timer=ps7_scutimer_0` 明确指定
- 脚本：`/mnt/d/workspace/zynq_pov/tools/fix_xiltimer.sh`

### xil_printf
- 不支持 `%lu`、`%f`、`%lx`。只支持 `%d %u %x %c %s %%`
- FPS 打印用整数毫秒 + 百分之一方式

### XSDB
- 下载 + 启动 CPU 的标准脚本：`tools/dl_helloworld.tcl`
- 运行在 Windows cmd.exe 环境下调用

### WSL 到 Windows 的桥接
- cmd.exe /c "..." 从 WSL 执行 Windows 工具
- 密集用的命令都加到了 `.claude/settings.local.json` 的 allow 列表
- bypassPermissions 模式不再弹框

### Vivado BD 集成（进行中）
- 脚本：`tools/bd_add_pov_ip.tcl`
- 架构：
  ```
  PS M_AXI_GP0 --[axi_smc 4M]--> pov_project.s_axi_control (0x43C20000)
  pov.m_axi_gmem0 (读模型 128b) -+
                                 +-[axi_smc_pov_hp 2S1M]-> PS S_AXI_HP1 -> DDR
  ```

- VDMA 保持走 HP0 不动（继续读 fb 送 HDMI）
- pov_project 跑 FCLK_CLK1=75 MHz（HLS 对 150MHz 优化, 75MHz 时序有 2× 余量）
- 地址：pov 控制 `0x43C20000`，pov 数据访问 DDR via HP1 (0..1GB)
