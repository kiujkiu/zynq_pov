### 4. UART baud 115200 → 921600 (#22)

发现根因:**ARM 自身改 BAUDGEN 寄存器写不进去** (BSP 某 reinit 路径回写默认值)。
解法:在 `dl_helloworld.tcl` 用 xsdb mwr 直接写 BAUDGEN/BAUDDIV (JTAG 级别),ps7_init 跑完后立即覆盖。

注意:**BAUDDIV 最小合法值=4** (per Zynq TRM)。若写 <4 寄存器拒绝,导致 BAUDGEN 改了但 BAUDDIV 不变 → 实际 baud 错。

最终值:BAUDGEN=18 (0x12), BAUDDIV=5 → 100M/(6×18) = 925926 (期望 921600,误差 0.5%)

Wire send 25 KB 帧时间:2.17 sec (115200) → 0.27 sec (921600),**8× 提速**。

### 5. cache flush 分块 (修 921600 下渲染散点)

主循环结尾的 `Xil_DCacheFlushRange(fb_write, 2.76 MB)` 一次性 ~2.7 ms,在 921600 baud 下 RX FIFO (64 byte) 0.69 ms 就溢出 → wire 字节丢失 → PPCL parser 仍按 byte count 完成但 model 被随机化 → 显示散点。

修法:64 KB 一块刷,块间插 `uart_poll_frame()`,RX FIFO 不溢。实测 100K 点干净到达。

### 6. HDMI 双缓冲撕裂修复 (#11)

VDMA park 模式下 `curR` 寄存器 (bits[20:16]) **永远不更新** —— 之前 DEBUG_LOG B6 标 "未解决"。

修法:**改 pure circular 模式**:
- 删 `XAxiVdma_StartParking` (park 不可靠)
- VDMA 自动每 vsync 切 buffer 0/1
- ARM 渲染前 poll curR 直到变化 → vsync-lock
- 写到 VDMA 不在读的 buffer

实测:FPS 113 自由跑 → 60.54 锁 60Hz vsync。撕裂消除。
副发现:`occupied_list` 不能放 `0x1B000000` 固定 DDR (BSP MMU 配置问题),回 BSS 数组 (1.5 MB) OK。

### 7. 1080p30 HDMI 升级 (#13)

`USE_1080P30` 编译宏切换。共用 74.25 MHz pixel clock。`XVtc_SetGeneratorTiming` 运行时改 timing struct,BD 不动。

实测:启用后 ARM 报 30 FPS = 1080p30 vsync 锁定。**直连 1080p 显示器可看**。但 USB-C HDMI capture 卡 (C1-1) 多限 720p 输入,demo 默认保留 720p。

### 8. host 渲染参数大全

`glb_to_points.py` + `anim_loader.py`:
- `lighting`: none / lambert / half-lambert / **studio** (三光 HDR-like 过曝)
- `ambient`: 0..1 float
- `z_stretch`: 默认 1.0,可上调让薄模型不在侧视消失
- `crop_top_frac`: 0..1,只保留模型顶 N% 然后重归一化 (头部聚焦用)
- `color_mode`: keep / height / random / uniform
- `brighten` / `gamma`

GUI 全部接入 spinner/dropdown,实时 re-sample。

### 9. PPCL 协议 num_points u16 → u32

支持单帧 100K-1M 点。HDR_FMT `<IIHHI` → `<IIIHH`,16 byte 头大小不变。host + board 头解析同步。

### 10. 内存重布局支持 1M 点

| 区域 | 地址 | 大小 |
|---|---|---|
| FB_A | 0x10000000 | 2.7 MB (720p) / 6 MB (1080p) |
| MODEL | 0x1A000000 | 16 MB (1M × 16B) |
| RING | 0x12000000 | 2.6 MB |
| FB_B | 0x14000000 | 同 FB_A |
| **VOXEL_GRID** | **0x18000000** | **32 MB (256³ × 2B)** |
| occupied_list | BSS | 1.5 MB (256K × 6B) |

DDR 上限 0x20000000 (512MB),用量 ~60 MB 才 12%。

## 关键 bug 修复

### B1: 921600 下 wire 散点
**原因:** 主循环 cache flush 32 MB 一块刷,期间 RX FIFO 溢出。
**修:** 分 64 KB 块刷 + 块间 UART poll。

### B2: VDMA park 模式 curR 不更新
**原因:** Xilinx VDMA park mode 实测 curR 寄存器永不更新。
**修:** 改 pure circular,ARM 端 poll curR vsync-lock。

### B3: 0x1B000000 写不持久
**原因:** BSP MMU/cache 配置区域可能未标 cacheable。
**修:** occupied_list 回 BSS 数组。

### B4: ARM 写 BAUDGEN 不持久
**原因:** BSP 某路径会回写默认。
**修:** xsdb dl 阶段 (JTAG 级) 改 baud,绕过 ARM。

### B5: BAUDDIV ≥4 才合法
**原因:** Zynq TRM 限制 BDIV ∈ [4, 255],<4 寄存器拒绝。
**修:** 选 BDIV=5 BGEN=18 给 921600。

### B6: 模型扁 90° 不可见
**原因:** anime/Hulkbuster 模型 z 范围远小于 x/y,Y 轴旋转 90° 投影变竖条。
**修:** 体素化 + cell 自带 3D extent;OR `z_stretch` 参数;OR `crop_top_frac` 头部聚焦。

### B7: 100K+ 点饱和
**现象:** 100K → 1M 视觉几乎不变,因为 256³ + cell=1 voxel 数已饱和。
**结论:** 真要更细节必须升级到 512³ (256 MB DDR,板子撑不住) 或 host 端预渲染切片图发板。
