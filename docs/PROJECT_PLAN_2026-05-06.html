1 · 一页总览

POV-3D 是一台旋转 LED 持久化显像（Persistence-of-Vision）3D 显示器。 主板：Zynq-7020 (XC7Z020)。Pipeline： 主机加载 .glb → 体素化（host or 设备） → ARM 渲染 / 未来 PL voxel slicer → HDMI 预览（720p/1080p）→ 旋转 LED 面板（PCB 未到货）。

稳定基线
60.5 FPS
USE_PL=0 + ARM render + vsync-lock @ 720p60
高细节模式
30.3 FPS
100K pts + 256³ voxel @ 720p60
主吞吐缺口
8×
~2800 vs 21600 slice/sec 目标
主线阻塞
2 项
USE_PL=1 m_axi 回归 + LED PCB 未到

本周一句话

展示路径已经稳了（颜色对齐 3D Viewer，HDMI 撕裂修了，UART 升级到 921600）。 瓶颈从"能不能看见"转到"是否能上 LED 面板"—— 需要把 pov_voxel_slice_batch IP 4 路并行集成到 BD，并修复 USE_PL=1 的 m_axi 写 DDR 回归。

7 · 任务卡片（详细）

P0 — 阻塞主线，必须先解

P0 T-20 USE_PL=1 时 PL m_axi 写 ring DDR 失败 Blocked
估时 0.5–2 day 前置 无 领域 Vivado BD / ChipScope References docs/claude_memory/feedback_use_pl1_uart_unreliable.md
现象：pov_project_batch_0 触发后 ap_done 正常，但 ring buffer 区 nz=0。同一 bitstream 之前有 nz=20060 工作过，现在回归。
怀疑：axi_smc_pov_hp/S00_AXI / S01_AXI 路由问题；或 pov_project_batch_0/m_axi_gmem* 路由被改坏。
调试步骤：
  1. Vivado 打开 BD，目视确认 axi_smc_pov_hp → processing_system7_0/S_AXI_HP1 仍连通。
  2. 加 ChipScope ILA 抓 pov_project_batch_0/m_axi_gmem0 的 AWVALID/AWADDR/WVALID。
  3. 若 AW 不发，看 ap_start 触发时 ap_idle 是否复位异常。
  4. 实在搞不定退回 USE_PL=0（baseline 完美）；hybrid mode 推迟。
验收：USE_PL=1 时收到 anime 后 ring nz ≥ 18 000。

P1 — 性能扩展，影响 LED 上线节奏

P1 T-30 4-IP voxel slicer Vivado BD 集成 + bitstream
估时 1 day（30–50 min 综合 × 多次迭代） 前置 T-20 解决（确认 m_axi 路由健康） 领域 Vivado BD / Tcl
已就绪：tools/bd_4ip_voxel_slicer.tcl、voxel_slicer_4ip.h、综合好的 pov_voxel_slice_batch IP。
关键风险：4 路 m_axi 都接 HP1 会撞带宽；需要做 HP0 / HP1 分流。先单 IP 跑通 → 加 IP 数 → 加 HP 端口。
验收：实测 slice/sec ≥ 8 000（4× 当前），FPS @ 720p ≥ 100，且模型颜色 / 几何与 ARM render 一致。
P1 T-10 hybrid mode：boot ARM 渲染 → 收 anime 切 PL In Progress
估时 0.5 day（依赖 T-20） 前置 T-20 领域 ARM C / cache 协议
已就绪：
  • read_model_n() noinline + DSB barrier 解决 cache stale read。
  • MMU 把 ring buffer 0x12000000 的 3 MB 段标 non-cacheable（属性 0x11DE2）。
  • main loop 根据 model_n > 1000 切 ARM ↔ PL 路径。
  • boot 期 ARM 渲染塞 100K busy poll 保证 anime UART 接收稳定。
未做：切到 PL 后 m_axi 写 DDR（被 T-20 阻）。
验收：boot 时 60 FPS ARM 渲染 + UART 100% 收完 30K 点 → 切 PL 后 ≥ 100 FPS。
